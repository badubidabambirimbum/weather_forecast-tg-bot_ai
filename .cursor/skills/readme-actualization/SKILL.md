# README Actualization

## Quick Start

1. Прочитать текущий `README.md`.
2. Сверить фактическое состояние проекта:
   - точки входа приложения;
   - команды запуска;
   - структуру директорий;
   - доступные тесты и команды их запуска.
3. Обновить `README.md` без выдуманных команд и без устаревших шагов.
4. Проверить, что в README есть обязательные разделы (см. чеклист ниже).

## Обязательные разделы README

При актуализации `README.md` всегда включать и держать в актуальном состоянии:

1. Pipeline запуска для пользователя:
   - prerequisites (Python/Node/зависимости, env-переменные);
   - пошаговый порядок запуска компонентов;
   - команды для локального запуска и базовой проверки, что все поднялось.
2. Структура проекта:
   - ключевые директории и их назначение;
   - основные entrypoints.
3. Описание функциональностей:
   - что умеет система на уровне пользователя;
   - кратко по ключевым сценариям.
4. Описание тестов:
   - какие типы тестов есть;
   - как запускать;
   - что считается успешным результатом.
5. Содержание (оглавление):
   - блок `Содержание` в начале документа;
   - якорные ссылки на ключевые разделы README.

## Workflow для агента

- Проверяй README на соответствие реальному коду, а не только на "красивый текст".
- Если команда запуска/теста не подтверждается проектом, уточняй или помечай как предположение.
- Не удаляй полезные рабочие инструкции без причины.
- Избегай дублирования: один источник истины для каждой команды.
- Пиши README на понятном русском языке, если пользователь не попросил другой язык.

## Чеклист перед завершением

- [ ] Есть актуальное описание функциональностей.
- [ ] Есть раздел по тестам с командами запуска.
- [ ] Есть блок `Содержание` с якорями на ключевые разделы.
- [ ] Команды и пути соответствуют текущему репозиторию.
